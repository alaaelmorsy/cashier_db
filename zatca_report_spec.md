## 4. حقول الفترة الزمنية

- **نوع الحقل:** `datetime-local` (تاريخ + وقت)
- **حقل "من":** id = `fromAt`
- **حقل "إلى":** id = `toAt`
- **تنسيق القيمة المُرسَلة للـ API:** `YYYY-MM-DD HH:MM:00`
  - مثال: `2026-04-09 00:00:00`
- **القيمة الافتراضية عند الفتح:** من = بداية اليوم الحالي 00:00، إلى = الوقت الحالي
- **نص الفترة:** يظهر أسفل الـ toolbar بعد الضغط على تطبيق بالشكل:
  ```
  الفترة: 2026-04-01 00:00:00 — 2026-04-09 23:59:00
  ```

## 5. القسم الأول — جدول الفواتير

**العنوان:** `الفواتير (مدفوعة وغير مدفوعة) ضمن الفترة`

### مصدر البيانات
- جلب جميع الفواتير من جدول `sales` حيث `created_at` يقع بين `date_from` و `date_to`
- **مهم:** الفلترة تعتمد على **تاريخ إنشاء الفاتورة** (`created_at`) وليس تاريخ الدفع
- استبعاد السجلات التي `doc_type = 'credit_note'` أو `invoice_no` يبدأ بـ `CN-`

### أعمدة الجدول

| # | اسم العمود | الحقل في البيانات | ملاحظة |
|---|---|---|---|
| 1 | رقم الفاتورة | `invoice_no` | |
| 2 | التاريخ | `created_at` | تنسيق: `DD/MM/YYYY hh:mm AM/PM` |
| 3 | العميل | `customer_phone` أو `customer_name` | يُعرض الهاتف إن وُجد وإلا الاسم |
| 4 | طريقة الدفع | `payment_method` | (نقدًا / شبكة / آجل / تمارا / تابي / تحويل بنكي / مختلط) |
| 5 | الحالة | محسوبة | **مدفوعة** (أخضر) / **غير مدفوعة** (أحمر) |
| 6 | المبلغ قبل الضريبة | `sub_total` | |
| 7 | الضريبة | `vat_total` | |
| 8 | الإجمالي | `grand_total` | |

### منطق تحديد الحالة
```
إذا كانت طريقة الدفع = 'credit' AND amount_paid <= 0
  → غير مدفوعة (خلفية حمراء فاتحة #fef2f2)
وإلا
  → مدفوعة (لا تغيير في الخلفية)
```

### صف الإجماليات (tfoot)
```
[إجمالي الفواتير] [colspan=5] | SUM(sub_total) | SUM(vat_total) | SUM(grand_total)
```

## 6. القسم الثاني — جدول المرتجعات

**العنوان:** `الفواتير المرتجعة (إشعارات دائنة) ضمن الفترة`

### مصدر البيانات
- جلب الإشعارات الدائنة حيث `created_at` يقع في الفترة المحددة
- **مهم:** الفلترة بـ `created_at` (تاريخ إنشاء الإشعار) وليس تاريخ آخر
- السجلات حيث `doc_type = 'credit_note'` أو `invoice_no` يبدأ بـ `CN-`
- **ملاحظة:** قيم `sub_total` و `vat_total` و `grand_total` قد تكون مخزّنة كأرقام **سالبة** في قاعدة البيانات — يجب استخدام `Math.abs()` عند العرض والحساب

### أعمدة الجدول

| # | اسم العمود | الحقل في البيانات |
|---|---|---|
| 1 | رقم الإشعار | `invoice_no` |
| 2 | التاريخ | `created_at` |
| 3 | العميل | `customer_phone` أو `customer_name` |
| 4 | رقم الفاتورة الأصلية | `base_invoice_no` أو `ref_base_invoice_no` |
| 5 | المبلغ قبل الضريبة | `Math.abs(sub_total)` |
| 6 | الضريبة | `Math.abs(vat_total)` |
| 7 | الإجمالي | `Math.abs(grand_total)` |

- لون خلفية الصفوف: `#fff7ed`

### صف الإجماليات (tfoot)
```
[إجمالي المرتجعات] [colspan=4] | SUM(abs(sub_total)) | SUM(abs(vat_total)) | SUM(abs(grand_total))
```

## 7. القسم الثالث — جدول المصروفات

**العنوان:** `المصروفات ضمن الفترة`

### مصدر البيانات
مزج مصدرين:
1. **مصروفات بسيطة** (`purchases`) — جدول عمليات الشراء البسيطة
2. **فواتير شراء** (`purchase_invoices`) — فواتير الموردين

كلاهما يُفلتر بـ `created_at` أو `purchase_at` في الفترة المحددة، ويُرتَّب تنازلياً بالتاريخ.

### أعمدة الجدول

| # | اسم العمود | الحقل في البيانات |
|---|---|---|
| 1 | البيان | `title` أو `name` (للبسيطة) / `"فاتورة شراء {رقم}"` للفواتير |
| 2 | التاريخ | `purchase_at` أو `created_at` |
| 3 | طريقة الدفع | `payment_method` (نقدًا / شبكة / آجل) |
| 4 | قبل الضريبة | `sub_total` |
| 5 | الضريبة | `vat_total` |
| 6 | الإجمالي | `grand_total` |

### صف الإجماليات (tfoot)
```
[إجمالي المصروفات] [colspan=3] | SUM(sub_total) | SUM(vat_total) | SUM(grand_total)
```

## 8. القسم الرابع — ملخص الإجماليات

**العنوان:** `ملخص الإجماليات`

### هيكل الجدول

| البيان | قبل الضريبة | الضريبة | بعد الضريبة |
|---|---|---|---|
| المبيعات | INV_PRE | INV_VAT | INV_GRAND |
| ناقص: المرتجعات | CN_PRE | CN_VAT | CN_GRAND |
| **المبيعات بعد خصم المرتجعات** | NET_SALES_PRE | NET_SALES_VAT | **NET_SALES_GRAND** |
| ناقص: المصروفات | EXP_PRE | EXP_VAT | EXP_GRAND |
| **الإجمالي النهائي** | **NET_PRE** | **NET_VAT** | **NET_GRAND** *(أخضر)* |

### معادلات الحساب

```
// المتغيرات الأساسية
INV_PRE   = مجموع sub_total   لجميع الفواتير
INV_VAT   = مجموع vat_total   لجميع الفواتير
INV_GRAND = مجموع grand_total لجميع الفواتير

CN_PRE    = مجموع Math.abs(sub_total)   للمرتجعات
CN_VAT    = مجموع Math.abs(vat_total)   للمرتجعات
CN_GRAND  = مجموع Math.abs(grand_total) للمرتجعات

EXP_PRE   = مجموع sub_total   للمصروفات
EXP_VAT   = مجموع vat_total   للمصروفات
EXP_GRAND = مجموع grand_total للمصروفات

// المبيعات بعد خصم المرتجعات
NET_SALES_PRE   = INV_PRE   - CN_PRE
NET_SALES_VAT   = INV_VAT   - CN_VAT
NET_SALES_GRAND = INV_GRAND - CN_GRAND

// الإجمالي النهائي
NET_PRE   = NET_SALES_PRE   - EXP_PRE
NET_VAT   = NET_SALES_VAT   - EXP_VAT
NET_GRAND = NET_SALES_GRAND - EXP_GRAND
```

### ⚠️ تنبيه مهم بشأن المرتجعات
قيم المرتجعات في قاعدة البيانات قد تكون **سالبة** (مثل `-11.50`).
يجب دائماً استخدام `Math.abs()` على قيم المرتجعات قبل أي عملية حسابية، وإلا سيُعطي الطرح نتيجة خاطئة:
```
// خطأ: 10 - (-10) = 20
// صحيح: 10 - Math.abs(-10) = 10 - 10 = 0
```

### ألوان صفوف الملخص

| الصف | الخلفية | ملاحظة |
|---|---|---|
| المبيعات | بلا / أبيض | عادي |
| ناقص: المرتجعات | `#fff7ed` | برتقالي فاتح |
| المبيعات بعد خصم المرتجعات | `#f0fdf4` + bold | أخضر فاتح |
| ناقص: المصروفات | `#fffbeb` | أصفر فاتح |
| الإجمالي النهائي (tfoot) | `#f1f5f9` | آخر عمود نص أخضر `#16a34a` |

## 9. تصدير PDF

### محتوى ملف PDF
يحتوي على نفس أقسام الشاشة بالترتيب:
1. ترويسة: عنوان التقرير + نص الفترة
2. جدول الفواتير (مع footer الإجماليات)
3. جدول المرتجعات (مع footer الإجماليات)
4. جدول المصروفات (مع footer الإجماليات)
5. جدول ملخص الإجماليات

### اسم الملف
```
zatca-report-{YYYY-MM-DD_HH-MM}--{YYYY-MM-DD_HH-MM}.pdf
```

### إعدادات الصفحة
- حجم: A4
- هوامش: 15px علوي/سفلي، 20px يمين/يسار
- اتجاه: عمودي (Portrait)

## 11. خلاصة منطق الفلترة

| القسم | حقل الفلترة | الشرط |
|---|---|---|
| الفواتير | `created_at` | بين `date_from` و `date_to` |
| المرتجعات | `created_at` | بين `date_from` و `date_to` |
| المصروفات | `purchase_at` أو `created_at` | بين `from_at` و `to_at` |

> **القاعدة الأساسية:** جميع الفلترة تعتمد على **تاريخ إنشاء السجل** وليس تاريخ الدفع. فاتورة آجل دُفعت خارج الفترة تظهر في الفترة التي أُنشئت فيها، ولا تظهر في فترة الدفع.
