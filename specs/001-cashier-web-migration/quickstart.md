# Quickstart: تشغيل والتحقق من cashier-web

## المتطلبات

- Node.js 18+
- MySQL يعمل وفيه قاعدة `cashier_db` الحالية (نفس قاعدة نسخة Electron)
- مجلد المشروع: `D:\PLUS\copy\cashier-web` (مجلد `cashier` الأصلي لا يُلمس)

## الإعداد والتشغيل

```powershell
cd D:\PLUS\copy\cashier-web
npm install
# .env: بيانات الاتصال بنفس القاعدة
#   DB_HOST=127.0.0.1  DB_PORT=3306  DB_USER=root  DB_PASS=...  DB_NAME=cashier_db  PORT=4300
npm start        # أو npm run dev أثناء التطوير
```

افتح المتصفح: `http://localhost:4300` (أو `http://<ip-الخادم>:4300` من أي جهاز على الشبكة).

## سيناريوهات التحقق

### 1. الأساس + تسجيل الدخول (User Story 1)
1. افتح التطبيق → تظهر صفحة تسجيل الدخول بنفس شكل نسخة سطح المكتب (عربي RTL).
2. ادخل بمستخدم موجود فعليًا في `cashier_db` → تصل للشاشة الرئيسية بصلاحياته.
3. بيانات خاطئة → نفس رسالة خطأ الأصل. فتح أي صفحة بدون دخول → إعادة توجيه لصفحة الدخول.

### 2. مطابقة صفحة منقولة (User Story 2 — يتكرر لكل صفحة)
1. افتح الصفحة في الويب وجنبها نفس الصفحة في نسخة Electron.
2. قارن: الشكل، الحقول، الأزرار، البيانات المعروضة، رسائل الأخطاء، والعمليات (إضافة/تعديل/حذف).
3. معايير القبول: راجع «عقد نقل الصفحة» في [contracts/http-api.md](./contracts/http-api.md) §6.
4. **الاعتماد النهائي من المستخدم** قبل بدء الصفحة التالية.

### 3. قاعدة بيانات مشتركة (User Story 3)
1. أنشئ فاتورة من الويب → افتح نسخة Electron → الفاتورة موجودة بنفس الرقم والتسلسل.
2. عدّل منتجًا من Electron → أعد تحميل صفحة المنتجات في الويب → التعديل ظاهر.
3. شغّل النسختين معًا وأنشئ فواتير من الاثنين → لا تكرار في أرقام الفواتير.

### 4. عدم المساس بالأصل (User Story 4)
```powershell
cd D:\PLUS\copy\cashier
git status   # يجب أن يكون نظيفًا (باستثناء specs/ الخاصة بهذه الميزة)
```
